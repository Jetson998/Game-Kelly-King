# MVP 规格说明

## 1. MVP 目标

做出一个浏览器可运行的最小版本，让玩家体验完整闭环：

```text
看拍品 → 竞价 → 成交 → 鉴定 → 出售 → 资金变化 → 进入下一天 → 结局
```

## 2. 技术范围

- 纯 HTML/CSS/JavaScript
- 不使用框架
- 不使用后端
- 不使用数据库
- 直接打开 `src/index.html` 可玩

## 3. 游戏参数

```text
初始现金：1000
库存上限：3
总天数：7
每天拍品数：5
胜利目标：10000
失败条件：现金 <= 0 且库存为空
```

## 4. 页面区域

### 顶部状态栏

显示：

- 当前天数
- 现金
- 库存数量
- 当前市场热点

### 拍品卡片

显示：

- 名称
- 品类
- 描述
- 品相
- 起拍价
- 模糊估值
- 风险提示

不直接显示真实价值。

### 竞价区域

显示：

- 当前价格
- 当前最高出价者
- 加价按钮：+50、+100、+500
- 放弃按钮

### NPC 区域

显示 3 个 NPC：

- 新手小白
- 二手贩子
- 抬价托

显示他们的最近行为。

### 日志区域

显示：

- 谁加价
- 谁放弃
- 成交结果
- 鉴定结果
- 出售结果

## 5. 拍品字段

```js
{
  id: 'old-camera-001',
  name: '老式胶片相机',
  category: '相机',
  condition: '七成新',
  description: '卖家说是家里翻出来的，不太懂。',
  startPrice: 200,
  realValue: 1800,
  risk: '品牌标识有磨损',
  rarity: 'rare'
}
```

## 6. NPC 字段

```js
{
  id: 'rookie',
  name: '新手小白',
  type: 'random',
  cashFactor: 0.8,
  accuracy: 0.4,
  aggression: 0.7
}
```

## 7. 拍卖规则

- 每件拍品从起拍价开始
- 玩家可以加价或放弃
- NPC 每轮根据心理价决定是否跟价
- 无人继续加价时成交
- 玩家最高价则扣现金并进入库存
- NPC 最高价则玩家错过该拍品

## 8. 鉴定规则

第一版简单处理：

- 玩家拍下后立即鉴定
- 显示真实价值
- 显示盈利/亏损预估

## 9. 出售规则

第一版只有快速出售：

```text
售价 = 真实价值 × 市场倍率 × 随机波动
```

市场倍率：

- 热门品类：1.2
- 普通品类：0.9 - 1.05

## 10. 结局

7 天结束后：

- 现金 >= 10000：胜利
- 现金 < 10000：挑战失败，但显示最终资产
