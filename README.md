# 捡漏之王

一个轻量网页小游戏：在 7 天流动拍场里，用 ￥1,200 起家，通过竞拍、鉴定、快速出售和读懂对手，把现金做到 ￥6,500。

当前版本偏「半现代江湖拍场」：货源来自华强北、坂田仓库、港货回流、澳门高端局散货、广州旧档口；对手则带一点江湖绰号和拍场心理战。

## 怎么玩

1. 看拍品的起拍价、模糊估值、风险和今日热点。
2. 观察三位对手动向：
   - **青衣少侠**：冲动，容易被稀罕物带节奏。
   - **铁算盘沈三**：算利润，价格不香就撤。
   - **笑面狐胡不归**：会抬价，不一定真想买。
3. 选择加价或收手观望。
4. 拍下后会立即鉴定真实价值，并进入库存。
5. 库存可快速出售；热点品类通常卖价更好。
6. 7 天、每天 5 件拍品后结算，现金达到 ￥6,500 即挑战成功。

## 当前版本状态

已完成 MVP 到发布前体验整理：

- 单机网页运行，无后端、无构建步骤。
- 30 个拍品数据，覆盖 5 类大湾区货源，并包含更明确的高风险/高波动样本。
- 3 个差异化 NPC，对应冲动、理性、抬价三种竞价行为。
- 每日市场热点与拍场风声，影响拍品出现、NPC 判断和快速出售报价。
- 7 天挑战、库存、鉴定、快速出售、结算。
- 挑战结束后会生成成就与复盘，包括最佳捡漏、最佳变现、最亏打眼、账面净利和下局策略建议。
- 加价、收手、落槌、成交、流拍有轻量动画、提示条和可关闭音效，支持系统减少动态效果设置。
- 首次进入会显示 3 步新手引导，并高亮现金目标、判断提示、主操作按钮；之后可在设置里重新打开。
- 每天随机出现拍场事件，例如现金客进场、查货风声、雨天冷场等，让同一套拍品每天节奏不同。
- localStorage 自动存档，刷新后可继续。
- 移动端布局、按钮触控区域、短文案和成交反馈已做一轮优化。

## 本地运行

最简单方式：直接用浏览器打开：

```text
src/index.html
```

如果浏览器对本地脚本有限制，建议在项目目录启动一个静态服务器：

```bash
python3 -m http.server 4173 -d src
```

然后访问：

```text
http://127.0.0.1:4173/
```

## 验证方式

```bash
node --check src/main.js src/auction.js src/data/items.js src/data/npcs.js
git diff --check
python3 -m http.server 4173 -d src
curl -fsS http://127.0.0.1:4173/ | grep '捡漏之王'
```

手动试玩建议检查：

- 首次进入是否显示 3 步新手引导，并依次高亮状态栏、判断提示、底部主操作；跳过/开始后不再自动弹出，设置里可重新打开。
- 首屏是否能看懂目标和基本玩法。
- 加价、放弃、下一件是否顺畅；加价价格板、落槌、成交提示和音效开关是否有反馈。
- 对手提示是否有参考价值但不剧透真实价值。
- 拍下后鉴定反馈是否明显。
- 刷新页面是否恢复进度。
- 7 天结束后是否出现称号、成交统计、最佳捡漏、最佳变现、最亏打眼和下局策略建议。
- 手机窄屏下按钮是否好点。

## 项目结构

- `src/index.html`：页面结构
- `src/styles.css`：视觉与响应式布局
- `src/main.js`：游戏状态、渲染、存档、流程控制
- `src/auction.js`：NPC 出价逻辑
- `src/data/items.js`：拍品数据
- `src/data/npcs.js`：NPC 数据
- `docs/`：策划、开发计划、任务清单和进度记录
