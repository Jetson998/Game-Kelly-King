# 湾区收买佬

一个轻量网页小游戏：在 7 天流动拍场里，用 ￥1,200 起家，通过竞拍、鉴定、快速出售和读懂对手，把现金做到 ￥6,500。打开页面即可试玩，单局约 8 分钟。

当前版本偏「半现代江湖拍场」：货源来自华强北、坂田仓库、港货回流、澳门高端局散货、广州旧档口；对手则带一点江湖绰号和拍场心理战。

## 怎么玩

1. 看拍品的起拍价、模糊估值、风险和今日热点。
2. 观察三位对手动向：
   - **青衣少侠**：冲动，容易被稀罕物带节奏。
   - **铁算盘沈三**：算利润，价格不香就撤。
   - **笑面狐胡不归**：会抬价，不一定真想买。
3. 选择加价或收手观望。
4. 拍下后会立即鉴定真实价值，并进入库存。
5. 库存可快速出售；热点品类通常卖价更好。
6. 7 天、每天 5 件拍品后结算，现金达到 ￥6,500 即挑战成功。

## 当前版本状态

已完成 MVP 到发布前体验整理：

- 单机网页运行，无后端、无构建步骤。
- 30 个拍品数据，覆盖 5 类大湾区货源，并包含更明确的高风险/高波动样本。
- 3 个差异化 NPC，对应冲动、理性、抬价三种竞价行为。
- 每日市场热点与拍场风声，影响拍品出现、NPC 判断和快速出售报价。
- 7 天挑战、库存、鉴定、快速出售、结算。
- 挑战结束后会生成成就与复盘，包括最佳捡漏、最佳变现、最亏打眼、账面净利和下局策略建议。
- 首页新增外部试玩入口式发布区，突出 7 天现金挑战、30 件拍品、3 位对手和“开始捡漏”按钮。
- 首页新增试玩反馈卡，可一键复制反馈模板，重点收集“哪里迷路、哪里上头、哪里想重来”。
- 拍品判断区新增“出价止损线”，根据估值、风险词、稀有度、热点和当前现金给出更明确的收手参考。
- 结算页新增战绩分享卡，可复制本局称号、现金、最终资产、成交数和最佳捡漏文案。
- 加价、收手、落槌、成交、流拍有轻量动画、提示条和可关闭音效，支持系统减少动态效果设置。
- 首次进入会显示 3 步新手引导，并高亮现金目标、判断提示、主操作按钮；首局前 2 件货会优先给低额、风险清楚的教学样本；之后可在设置里重新打开教程。
- 每天随机出现拍场事件，例如现金客进场、查货风声、雨天冷场等，让同一套拍品每天节奏不同。
- localStorage 自动存档，刷新后可继续。
- 小屏首屏压缩模式：手机视口会隐藏发布介绍与反馈卡，改用吸顶轻导航、横滑状态卡、价格优先拍品卡和固定底部四按钮，让状态、当前价、拍品、止损线与主操作更快进入视野。

## 本地运行

最简单方式：直接用浏览器打开：

```text
src/index.html
```

如果浏览器对本地脚本有限制，建议在项目目录启动一个静态服务器：

```bash
python3 -m http.server 4173 -d src
```

然后访问：

```text
http://127.0.0.1:4173/
```

```bash
node --check src/main.js src/auction.js src/data/items.js src/data/npcs.js
git diff --check
python3 -m http.server 4173 -d src
curl -fsS http://127.0.0.1:4173/ | grep '湾区收买佬'
```

手动试玩建议检查：

- 首次进入是否显示 3 步新手引导，并依次高亮状态栏、判断提示、底部主操作；跳过/开始后不再自动弹出，设置里可重新打开。
- 新开一局的前 1-2 件货是否更适合练手，不会一上来随机到高价重仓或强陷阱样本。
- 首屏是否能看懂目标和基本玩法，并能通过“开始捡漏”直接进入拍卖主舞台。
- 拍品区是否能看到“止损线”，并随当前价变化为安全、警告或收手提示。
- 加价、放弃、下一件是否顺畅；加价价格板、落槌、成交提示和音效开关是否有反馈。
- 对手提示是否有参考价值但不剧透真实价值。
- 拍下后鉴定反馈是否明显。
- 刷新页面是否恢复进度。
- 7 天结束后是否出现称号、成交统计、最佳捡漏、最佳变现、最亏打眼、下局策略建议和可复制战绩文案。
- 点击首页“复制反馈模板”后，是否能复制包含当前进度的反馈问题清单。
- 手机窄屏下是否直接进入压缩模式：首屏能看到状态、当前价、拍品、止损线和底部主按钮，库存/日志可通过快捷入口打开。

## 试玩反馈建议

如果要找朋友试玩，建议只问这 5 个问题，避免泛泛地问“好不好玩”：

1. 第一次打开时，是否立刻知道目标和下一步要点？
2. 哪个环节最卡/最迷路：看货、出价、对手、鉴定、库存、结算？
3. 哪一刻最有“捡漏 / 上头 / 后悔”的感觉？
4. 手机上按钮、文案、信息量是否清楚？
5. 如果只改一件事，最希望改什么？

下一轮优先级：先修“迷路点”和移动端点击障碍，再调数值，最后再扩内容。
